Interface Normalization

Denormalized vs. Normalized Interface Design

Below are links to a simplistic example of normalizing a software interface. The scenario is a simple contact and contact note manager.

Denormalized Contact Manager
Total Screens: 3 / Benefits: Low amount of navigation / Cons: Learning required, Higher level of Confusion, Slower to read, Slower to scan, Little Room for Self-Documentation

Normalized Contact Manager
Total Screens: 12 / Benefits: Self-evident, Quickly Readable, Quickly Scannable, Low Confusion, Room for Self-Documentation / Cons: Increased navigation

Features Represented:

The interface includes the following features:

  • Entry Point
  • Create New Contact
  • Find a Contact
  • Edit a Contact
  • Delete a Contact
  • Add a Note
  • Review a Contact's Notes
  • Report on All Notes By Day
